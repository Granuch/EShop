## 📖 Книги

### Microservices Architecture

#### 1. **Building Microservices** by Sam Newman
- **Level**: Beginner to Intermediate
- **Topics**: Microservices principles, communication patterns, deployment
- **Link**: [Amazon](https://www.amazon.com/Building-Microservices-Designing-Fine-Grained-Systems/dp/1492034029)
- **Why**: Найкраща книга для старту з мікросервісами

#### 2. **Microservices Patterns** by Chris Richardson
- **Level**: Intermediate to Advanced
- **Topics**: Saga, CQRS, Event Sourcing, API Gateway patterns
- **Link**: [Amazon](https://www.amazon.com/Microservices-Patterns-examples-Chris-Richardson/dp/1617294543)
- **Why**: Детальний розбір patterns з прикладами

#### 3. **.NET Microservices: Architecture for Containerized .NET Applications**
- **Level**: Intermediate
- **Topics**: .NET-specific microservices, Docker, Kubernetes
- **Link**: [Free eBook](https://dotnet.microsoft.com/learn/aspnet/microservices-architecture)
- **Why**: Офіційний guide від Microsoft з eShopOnContainers

---

### Domain-Driven Design (DDD)

#### 4. **Domain-Driven Design** by Eric Evans (Blue Book)
- **Level**: Advanced
- **Topics**: Bounded Contexts, Aggregates, Entities, Value Objects
- **Link**: [Amazon](https://www.amazon.com/Domain-Driven-Design-Tackling-Complexity-Software/dp/0321125215)
- **Why**: Класика, must-read для DDD

#### 5. **Implementing Domain-Driven Design** by Vaughn Vernon (Red Book)
- **Level**: Intermediate
- **Topics**: Practical DDD implementation, code examples
- **Link**: [Amazon](https://www.amazon.com/Implementing-Domain-Driven-Design-Vaughn-Vernon/dp/0321834577)
- **Why**: Практичніший за Blue Book, багато прикладів

#### 6. **Domain-Driven Design Distilled** by Vaughn Vernon
- **Level**: Beginner
- **Topics**: DDD за 100 сторінок
- **Link**: [Amazon](https://www.amazon.com/Domain-Driven-Design-Distilled-Vaughn-Vernon/dp/0134434420)
- **Why**: Швидкий огляд DDD concepts

### Clean Architecture

#### 7. **Clean Architecture** by Robert C. Martin (Uncle Bob)
- **Level**: Intermediate
- **Topics**: SOLID principles, dependency inversion, layers
- **Link**: [Amazon](https://www.amazon.com/Clean-Architecture-Craftsmans-Software-Structure/dp/0134494164)
- **Why**: Fundamental knowledge для proper architecture

#### 8. **Clean Code** by Robert C. Martin
- **Level**: Beginner to Intermediate
- **Topics**: Readable code, naming, functions, comments
- **Link**: [Amazon](https://www.amazon.com/Clean-Code-Handbook-Software-Craftsmanship/dp/0132350882)
- **Why**: Основи якісного коду
